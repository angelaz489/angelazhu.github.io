Facebook Messenger Redesign

This project was started as a team project for a Usability and Information Architecture class at UCSD. After the completion of the project, I redesigned a couple other features of the application, based off the results from the user testing, and came up with a prototype encompassing all the redesigns.

UX Researcher, UX Designer

User Research, Competitive Analysis, Information Architecture, Prototyping, Testing, Wireframing, Figma

Dec 2019 (3 weeks)

Challenge

Redesign components of the Facebook Messenger application to increase visibility of functions and icons.

Deliverables

High fidelity prototypes illustrating redesigned interactions.

Background —

Facebook Messenger is an instant messaging platform, that is connected with the Facebook social media platform. The messaging system supports many features such as direct messaging, calling, location sharing, business interactions, third party app integration, instant games, and much more.

The Facebook Messenger application is comprised of 3 tabs:

  • Chats

    List of current individuals and group chatrooms.
    Add and stories view stories posted by friends.
  • People

    List of Facebook friends who the user has interacted with most frequently or recently.
    Add stories and view stories posted by friends.
  • Discover

    Access to instant games or ability to message businesses on Facebook.

Research —

In order to understand what components of the Facebook Messenger app were lacking, my team came up with a couple pre and post interview questions and tasks to conduct user testing interviews on the existing application. The people we chose to interview were existing users who were either acquainted or familiar with the application.

We asked the pre interview questions to gauge the interviewees initial understanding and experience with the app. We aimed to tailor the tasks so that the user could interact with as much of the features on the application as possible. Although we did physically observe the user, we added the post interview questions to let the user talk about their experience and maybe provide us with details we could have missed during observation.

Findings and Trends—

While we did not see a specific pattern in the outcome of the tasks (successful/unsuccessful), many users did explain to us how they were mainly using the process of elimination to complete the tasks:

  • Users were unable to find many functions and had difficulty navigating the different pages because of the overloaded content and abundance of features.
  • All of our users did not understand the 'Discover' icon and contents on the page.
  • Users had difficulty finding message requests sent from people they were not Facebook friends with.

Problems —

  1. Overloaded content and abundance of features
  2. Unclear ‘Discover’ icon and page
  3. Visibility of message requests

Competitive Analysis —

Instagram Direct Messaging

  • Messaging system incorporated into main social media application allowing for a less cluttered and simple page solely for messaging.
  • When users on Instagram receive a message from someone they are not following, they simply receive the message, unlike Facebook where the users needs to first approve the message request on a separate page.

Snapchat

  • Messaging system is the main aspect, while also featuring a Discover page for any Snapchat media shows, subscriptions to media shows or verified people/accounts, and Snapchat stories from friends.
  • Does not allow for users to send or receive messages to or from anyone who is not their friend.
  • Chat and Discover pages are clearly separated and signified using both icon and text.

Apple iMessage

  • Unlike Facebook Messenger, iMessage is not connected to any social media platforms and its only purpose is for chatting.
  • Like Messenger, users can receive messages from strangers (non-friends). However the message is shown along with the all other chats unlike Facebook Messenger where messages from non-friends are contained within a different page.

Sketches and Ideation —

Redesign —

  1. Overloaded content and abundance of features.

    • Removed stories from ‘Chats’ page and kept the already existing stories on ‘People’ page.

  2. Unclear ‘Discover’ icon and page.

    • Changed ‘Discover’ icon to new icon comprised of both ‘Pages’ and ‘Gaming’ icon from Facebook platform.

    • Added ‘Games’ tab in ‘Discover’ page to further organize the content and increase efficiency.

  3. Visibility of message requests.

    • Moved message requests to ‘Chats’ page and made it readily available.

    • Added ‘...’ icon to top right of “Chats’ page to contain any other content from previous ‘Message Request’ icon (Eg. filtered messages).

Prototype —

I would like to take my prototype further by conducting more user tests to figure out the effectiveness of my redesigns. If I had more time, I would have conducted the user tests on a larger sample of users and on a more diverse pool of interviewees (non-students).
